# De-AI the Copy -- OnlyClaw Publish Readiness Pass 2

This document identifies every place the platform reads like it was written by AI, and provides specific fixes. The goal: make OnlyClaw feel like it was built by humans with opinions, not generated by a language model with a style guide.

---

## The Core Problems

### 1. The Eyebrow Epidemic
Every single section on every single page has a tiny uppercase eyebrow label above the heading. There are **150+ instances** of `text-xs font-semibold uppercase tracking-[0.28em]` eyebrows across the app. Real sites use them sparingly for emphasis. Here, they're visual noise.

**Pattern**: `eyebrow → h2 → description paragraph → cards/content`
**Appears on**: Every page, every section, every card. The entire app is a stack of identical blocks.

### 2. Every Number Explains Itself
Metric cards across the app have a `detail` prop that adds a paragraph under every single number. Real dashboards show a number and a label. OnlyClaw explains what a number *means* in a sentence -- which screams "an AI was trying to be helpful."

### 3. Filler Phrase Bingo
These phrases appear repeatedly and are classic AI tells:
- "in one place" (fan home, creator dashboard, posts, subscribers)
- "at a glance" (fan home, billing)
- "seamless" / "effortlessly" (landing page benefits)
- "you know exactly" (landing page, creator profile)
- "designed to" (landing page)
- "from one [noun]" (creator dashboard, posts page, subscribers)
- "Browse, subscribe, and enjoy" (landing page)

### 4. Everything Gets a Description
There is no section without an explanatory paragraph. Even a grid/list toggle says "Switch between card grid and list view." Real sites trust their users. AI-generated sites explain everything.

### 5. Every Creator Bio is the Same Shape
All three demo creator bios follow the exact same structure: `"[Role] [activity] [list of three things], and [exclusive thing] for subscribers."` They read like they came from one prompt.

### 6. "Your" / "You can" Overload
Almost every piece of text addresses the user directly. Counted 40+ instances of "your" and 15+ of "you can" in user-facing copy. It creates a patronizing, hand-holding tone.

### 7. Status Badges on Everything
Feed items show 3 badges each ("Included with membership" + context label + "Ready in messages"). Creator cards show approval + verification + adult content badges. Badges are helpful; this many badges are noise.

### 8. Identical Empty States
Every empty state is a long paragraph explaining what will appear and when, followed by buttons. They all sound the same. Real empty states are one punchy sentence.

## Structural Fixes (Not Just Copy)

### A. Kill Most Eyebrows
Remove the eyebrow from at least 80% of sections. Reserve for:
- Page-level identity (one per page, max)
- Status indicators where the label adds real info
- Section breaks in very long pages

Affected component: `src/components/shared/page-header.tsx`, `FanPageHeader`, `CreatorPageHeader`, `AdminPageHeader`, and dozens of inline eyebrows.

### B. Remove MetricCard `detail` Prop Usage
The `detail` prop in `src/components/shared/metric-card.tsx` should be removed from most callsites. A number + label is enough. Only keep `detail` when the value genuinely needs context (e.g., "Next renewal: Mar 30").

### C. Trim Section Headers
Many sections have eyebrow + h2 + description. Most should have just an h2 (or nothing, if the content is obvious). The fan feed doesn't need a heading that says "Subscriber content and previews."

### D. Simplify Feed Item Badges
Each fan feed item currently shows 3 badges. Reduce to 1 (the access status). The context label can be regular text, not a badge.

### E. Give Creators Different Voices
The three demo creators should not sound like they were written by the same person. Each bio, headline, and highlight should have a distinct tone, length, and structure.

### F. Shorten All Empty States
Every empty state should be max 1 short sentence + 1 CTA button. No paragraphs. No "once X happens, Y will appear here."

### G. Remove Instructional Text from Forms
Form labels and layouts should be self-explanatory. Remove paragraphs that explain what form fields do or how to use the page.

## Priority Order

1. **Structural**: Kill eyebrows, remove metric details, simplify badges (affects every page)
2. **Landing + Discover + Profile**: These are the first pages users see
3. **Creator bios and demo data**: The "sameness" is very visible
4. **Fan dashboard + billing**: Over-explained
5. **Creator dashboard + messages + posts + subscribers**: Internal-facing but still too AI
6. **Admin pages**: Lower priority since only admins see these
7. **Error/empty states**: Quick wins
